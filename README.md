# cascade-js

A TypeScript implementation of CASCADE (Contextual AST-based Code Approach to Semantically Deobfuscate JavaScript), based on the research paper ["CASCADE: Contextual AST-based Code Approach to Semantically Deobfuscate JavaScript"](https://arxiv.org/abs/2507.17691v2) by Google.

CASCADE uses LLM-powered analysis combined with AST transformations to reverse JavaScript obfuscation, particularly targeting patterns generated by [javascript-obfuscator](https://github.com/javascript-obfuscator/javascript-obfuscator) and similar tools.

## Installation

```bash
pnpm add cascade-js
# or
npm install cascade-js
# or
yarn add cascade-js
```

## Quick Start

```typescript
import { deobfuscate, OpenAILLMAdapter } from 'cascade-js';

const obfuscatedCode = `
  var _0x1234 = ['Hello', 'World'];
  function _0x5678(n) { return _0x1234[n]; }
  console.log(_0x5678(0) + ' ' + _0x5678(1));
`;

const result = await deobfuscate(obfuscatedCode, {
  llmAdapter: new OpenAILLMAdapter(process.env.OPENAI_API_KEY),
  timeout: 60000,
});

console.log(result.code);
// Output: Clean, deobfuscated JavaScript
```

## CLI Usage

The CLI provides a convenient way to deobfuscate files from the command line.

```bash
# Basic usage
cascade deobfuscate input.js -o output.js

# With specific LLM provider
cascade deobfuscate input.js --provider openai --model gpt-4o-mini -o output.js

# Using environment variables for API keys
cascade deobfuscate input.js --provider anthropic -o output.js

# With custom timeout
cascade deobfuscate input.js --timeout 120000 -o output.js

# Debug mode for verbose output
cascade deobfuscate input.js --debug -o output.js
```

### CLI Options

| Option | Description | Default |
|--------|-------------|---------|
| `-o, --output <file>` | Output file path | stdout |
| `--provider <name>` | LLM provider: openai, anthropic, gemini, ollama | openai |
| `--model <model>` | Model name (provider-specific) | Provider default |
| `--timeout <ms>` | Timeout in milliseconds | 60000 |
| `--debug` | Enable debug logging | false |
| `--skip-prefilter` | Skip obfuscation detection | false |

## LLM Provider Setup

CASCADE supports multiple LLM providers for prelude detection. Choose the one that fits your needs:

### OpenAI

```typescript
import { OpenAILLMAdapter } from 'cascade-js';

const adapter = new OpenAILLMAdapter(
  process.env.OPENAI_API_KEY,
  'gpt-4o-mini' // optional, default: gpt-4o-mini
);
```

**Recommended models:**
- `gpt-4o-mini` (default) - Fast and cost-effective
- `gpt-4o` - Higher accuracy for complex obfuscation

### Anthropic (Claude)

```typescript
import { AnthropicLLMAdapter } from 'cascade-js';

const adapter = new AnthropicLLMAdapter(
  process.env.ANTHROPIC_API_KEY,
  'claude-3-haiku-20240307' // optional, default: claude-3-haiku-20240307
);
```

**Recommended models:**
- `claude-3-haiku-20240307` (default) - Fast and affordable
- `claude-3-sonnet-20240229` - Better accuracy
- `claude-3-opus-20240229` - Best accuracy

### Google Gemini

```typescript
import { GeminiLLMAdapter } from 'cascade-js';

const adapter = new GeminiLLMAdapter(
  process.env.GOOGLE_API_KEY,
  'gemini-1.5-flash' // optional, default: gemini-1.5-flash
);
```

**Recommended models:**
- `gemini-1.5-flash` (default) - Fast responses
- `gemini-1.5-pro` - Higher accuracy

### Ollama (Local)

For offline use or cost savings, run models locally with Ollama:

```typescript
import { OllamaLLMAdapter } from 'cascade-js';

const adapter = new OllamaLLMAdapter(
  'llama3.2', // model name
  'http://localhost:11434' // host (optional)
);
```

**Setup:**
1. Install Ollama: https://ollama.com
2. Pull a model: `ollama pull llama3.2`
3. Start Ollama server: `ollama serve`

**Recommended models:**
- `llama3.2` (default) - Good balance of speed and accuracy
- `codellama` - Optimized for code tasks

```typescript
import { MockLLMAdapter } from 'cascade-js';
```


## Custom Passes

You can extend CASCADE with custom transformation passes:

```typescript
import { deobfuscate, definePass, type Pass } from 'cascade-js';
import { parse } from '@babel/parser';
import generate from '@babel/generator';
import traverse from '@babel/traverse';

// Define a custom pass
const myCustomPass = definePass({
  name: 'my-custom-pass',
  dependencies: ['string-replacement'], // Runs after string-replacement
  
  async transform(code: string, context) {
    const ast = parse(code, { sourceType: 'script' });
    
    // Your transformation logic here
    traverse(ast, {
      // AST visitors...
    });
    
    return generate(ast).code;
  },
});

// Use custom passes
const result = await deobfuscate(code, {
  llmAdapter: adapter,
  customPasses: [
    stringReplacementPass,
    constantPropagationPass,
    myCustomPass,
  ],
});
```

### Pass Interface

```typescript
interface Pass {
  /** Unique name */
  name: string;

  /** Passes that must run before this one */
  dependencies: string[];

  /** Transform function */
  transform: (code: string, context: PipelineContext) => Promise<string>;
}
```

### Built-in Passes

| Pass | Description | Dependencies |
|------|-------------|--------------|
| `stringReplacementPass` | Replaces decoded strings from prelude | `constantPropagation` |
| `constantPropagationPass` | Propagates constant values | - |
| `inliningPass` | Inlines simple function calls | - |
| `cleanupPass` | Removes dead code and cleans up | `stringReplacement` |

## Architecture Overview

```
CASCADE Pipeline Flow
=====================

Input: Obfuscated JavaScript Code
                |
                v
        +-------------------+
        |  1. Pre-filter    |
        |  (Detection)      |  --> Quick heuristic check for obfuscation patterns
        +-------------------+
                |
                v
        +-------------------+
        |  2. Splitter      |
        |  (Parse & Tag)    |  --> Parse AST, tag statements with IDs
        +-------------------+
                |
                v
        +-------------------+
        |  3. LLM Analysis  |
        |  (Prelude Detect) |  --> LLM identifies string array, fetcher, rotate function
        +-------------------+
                |
                v
        +-------------------+
        |  4. Sandbox       |
        |  (String Extract) |  --> Execute rotate function safely, extract decoded strings
        +-------------------+
                |
                v
        +-------------------+
        |  5. Pipeline      |
        |  (AST Passes)     |  --> Apply transformations: string replacement, propagation, cleanup
        +-------------------+
                |
                v
Output: Deobfuscated JavaScript Code


Key Components
==============

1. Prefilter (detector.ts)
   - Pattern-based obfuscation detection
   - Hex identifier counting (_0x[a-f0-9]+)
   - String array pattern matching
   - Confidence scoring (0-1)

2. Statement Splitter (splitter.ts)
   - Babel parser integration
   - Statement boundary detection
   - ID assignment for LLM reference

3. Prelude Orchestrator (orchestrator.ts)
   - LLM adapter abstraction
   - Prompt engineering for pattern detection
   - Structured output parsing

4. Sandbox (executor.ts)
   - QuickJS-emscripten isolation
   - Safe execution of rotate functions
   - String extraction from prelude

5. Transformation Passes (passes/*.ts)
   - Babel-based AST transformations
   - Dependency-ordered execution
   - Context sharing between passes
```

## API Reference

### Core Functions

#### `deobfuscate(code, options?)`

Main entry point for deobfuscation.

```typescript
import { deobfuscate, OpenAILLMAdapter } from 'cascade-js';

const result = await deobfuscate(obfuscatedCode, {
  llmAdapter: new OpenAILLMAdapter(apiKey),
  timeout: 60000,
  skipPrefilter: false,
});

console.log(result.code);
console.log(result.stats);
```

#### `detectObfuscation(code)`

Quick detection without full deobfuscation.

```typescript
import { detectObfuscation } from 'cascade-js';

const detection = detectObfuscation(code);
console.log(detection.detected);    // boolean
console.log(detection.confidence);  // 0-1
console.log(detection.patterns);    // matched patterns
```

#### `createPipeline(config)`

Build custom pipelines.

```typescript
import { createPipeline, stringReplacementPass } from 'cascade-js';

const pipeline = createPipeline({
  options: { timeout: 30000 },
  passes: [stringReplacementPass],
});

const result = await pipeline.run(code);
```
